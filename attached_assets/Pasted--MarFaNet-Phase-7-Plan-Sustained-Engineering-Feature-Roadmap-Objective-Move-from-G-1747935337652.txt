# **MarFaNet — Phase 7 Plan: Sustained Engineering & Feature Roadmap**

> *Objective:* Move from GA launch into a stable maintenance mode while planning incremental innovations that keep MarFaNet competitive over the next 12 months.

---

## 1 • Maintenance Cadence (Sustained Engineering)

| Release Type      | Branch                           | Frequency                       | Typical Content                                                  |
| ----------------- | -------------------------------- | ------------------------------- | ---------------------------------------------------------------- |
| **Patch (1.0.x)** | `hotfix/1.0.x` → `release/1.0.x` | As‑needed (≤ 14 days after bug) | Blocker/Major bug fixes, security patches, minor perf tweaks     |
| **Minor (1.x.0)** | `develop`                        | Every 8–10 weeks                | New backward‑compatible features, UI/UX polish, protocol updates |
| **Major (2.0.0)** | `feature/2.0`                    | \~ Q2 2026                      | Architecture changes, breaking API updates, large design refresh |

### 1.1 SLA Targets

* **Security Hotfix:** ≤ 48 h from CVE disclosure
* **Crash Regression:** ≤ 72 h from detection
* **User‑reported Major:** ≤ 7 days triage, ≤ 21 days fix

## 2 • Observability & Ops Enhancements

1. **Prometheus Exporter** for core metrics (connections, RTT, failures) via gRPC endpoint.
2. **Grafana Dashboard** template for real‑time marfanet telemetry.
3. **Centralised Log Aggregation** to Datadog with custom dashboards.
4. **Automated Play‑Store Review Scraper** → sentiment trend alerts (weekly).
